About The Project

Non-invasive AI screening for Jaundice (eye sclera), Skin Diseases (face), and Nail Disorders — powered by deep learning with MobileNetV2 & EfficientNet transfer learning.

🎯

Project Goal

Build a non-invasive, AI-powered health screening platform that uses deep learning to analyze images of eyes, face skin, and nails to detect potential health conditions — without blood tests or lab visits.

👁️

Eye / Jaundice Detection

Jaundice is caused by elevated bilirubin levels, which turn the sclera (white of the eye) yellow. Our model analyzes eye images using Haar cascade eye detection and MobileNetV2 binary classification to detect this discoloration for early screening — without blood tests.

🧑

Face / Skin Disease Detection

Classifies facial skin conditions into 5 categories — Acne, Eczema, Herpes, Panu (Tinea Versicolor), and Rosacea — using EfficientNetB0 transfer learning for preliminary dermatological assessment.

💅

Nail Disease Detection

Identifies nail disorders including Psoriasis, Onychomycosis (fungal infection), and Healthy nails from photographs using MobileNetV2, enabling early detection of health indicators through nail analysis.

Problem Statement

Why this project was needed — addressing gaps in accessible health screening.

🏥

Limited Rural Access

Access to dermatological and ophthalmological screening is limited in rural and underserved areas. Many patients have to travel long distances for basic health assessments.

💉

Invasive Tests Required

Jaundice detection traditionally requires blood tests (bilirubin levels). Our system enables non-invasive screening through sclera image analysis — no needles required.

👨‍⚕️

Specialist Shortage

Common skin conditions like Acne, Eczema, Herpes, and Rosacea require dermatological expertise. AI can assist in preliminary classification where specialists aren't available.

🔍

Early Detection Gap

Nail abnormalities can indicate systemic health issues (liver disease, anemia, autoimmune conditions). Early detection through image analysis prompts timely medical consultation.

Project Objectives

The key goals we set out to achieve with HealthVision AI.

🎯 Core Objectives

  1. Develop a multi-module AI platform capable of analyzing eye, face, and nail images for disease detection.
  2. Implement transfer learning using MobileNetV2 and EfficientNetB0 architectures for high-accuracy classification with limited training data.
  3. Build an intelligent eye detection pipeline using OpenCV Haar cascades with dual-strategy detection and fallback mechanisms.
  4. Create a responsive web interface supporting both image upload and real-time camera capture with ROI guide overlay.
  5. Ensure privacy by processing all data locally — no images stored or transmitted to third-party services.
  6. Provide detailed disease information alongside predictions for educational and screening purposes.

✅ Key Deliverables

  1. Three trained deep learning models (.h5 format) for eye, face, and nail detection.
  2. Flask-based REST API backend with prediction endpoints for each module.
  3. Responsive single-page web application with camera and upload support.
  4. Comprehensive disease information database with 20+ conditions.
  5. Image enhancement pipeline (denoising, contrast normalization) for real-world images.
  6. Complete project documentation and deployment instructions.

Jaundice Detection

Eye Sclera Analysis

Nail Image Analysis

Training Datasets

Each detection module is trained on a curated medical image dataset, preprocessed to 224×224 RGB format.

👁️

Jaundice Eye Dataset

ezyZip Data Set

Classes: 2 (Jaundice, Normal)
Training Images: 136 | Validation: 32
Split: 80% Training / 20% Validation
Preprocessing: Rescale pixel values to [0, 1] (÷ 255)

Binary Classification 224×224 RGB 80/20 Split
🧑

Face Skin Disease Dataset

FaceZip Data Set

Classes: 5 (Acne, Eksim/Eczema, Herpes, Panu, Rosacea)
Training Images: 1,196 | Validation: 298
Split: 80% Training / 20% Validation
Preprocessing: EfficientNet preprocess_input (scales to [-1, 1])

5-Class Classification 224×224 RGB 80/20 Split
💅

Nail Disease Dataset

nailZip Data Set

Classes: 3 (Healthy, Onychomycosis, Psoriasis)
Training Images: 933 | Validation: 231
Split: 80% Training / 20% Validation
Preprocessing: Rescale pixel values to [0, 1] (÷ 255)

3-Class Classification 224×224 RGB 80/20 Split
Data Augmentation (Training): Rotation (20°), Width/Height Shift (15%), Zoom (20%), Horizontal Flip, Brightness Range [0.7–1.3], Shear (10%), Fill Mode: Nearest

Training Results & Accuracy

All models use a two-stage training strategy: frozen base → fine-tuning with lower learning rate.

Model Architecture Validation Accuracy Loss Function Train Images Val Images
Jaundice (Eye) MobileNetV2 90.62% Binary Cross-Entropy 136 32
Face (Skin) EfficientNetB0 72.15% Categorical Cross-Entropy 1,196 298
Nail MobileNetV2 85.71% Categorical Cross-Entropy 933 231
Two-Stage Training: Stage 1 — Base frozen, learning rate 1e-3, 15 epochs | Stage 2 — Top layers unfrozen, learning rate 1e-5, 25 epochs. Callbacks: EarlyStopping (patience=5), ReduceLROnPlateau (patience=3, factor=0.3), ModelCheckpoint (save_best_only).

Image Enhancement Pipeline

Every uploaded image passes through an enhancement pipeline before AI analysis for better real-world accuracy.

🔇

Step 1: Bilateral Denoising

Removes noise while preserving edges using bilateral filter (d=5, sigmaColor=50, sigmaSpace=50). This cleans up camera artifacts without blurring important features.

🌗

Step 2: CLAHE Normalization

Contrast Limited Adaptive Histogram Equalization applied on L channel of LAB color space (clipLimit=2.0, tileGrid=8×8). Normalizes lighting across different environments.

📐

Step 3: Resize & Normalize

Image resized to 224×224 pixels, converted BGR→RGB, and pixel values normalized (÷255 or EfficientNet preprocess) before model inference.
